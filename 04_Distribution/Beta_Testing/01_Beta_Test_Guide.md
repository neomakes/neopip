# 🧪 Beta Test Guide & Scenarios

이 문서는 베타 테스트의 전략, 실행 시나리오, 그리고 버그 리포팅 양식을 통합하여 관리합니다.

---

## 1. Beta Strategy (전략)

### 🎯 목표
1. **사용성 검증**: 핵심 기능(온보딩, 기록)의 UX 마찰 지점 식별
2. **데이터 품질 확인**: 수집된 데이터(로그)가 정상적으로 쌓이는지 확인
3. **가치 검증**: 사용자가 "자신을 이해한다"는 가치를 느끼는지 피드백 수집

### 🗓 일정 및 대상
* **기간**: 2026.01.20 - 2026.01.27 (1주)
* **대상**: 내부 팀원 및 지인 5명 (iPhone 사용자)
* **빌드**: TestFlight v1.0.0 (Build 5)

## 2. Test Scenarios (테스트 시나리오)

테스터에게 아래 시나리오를 수행하도록 요청하세요.

### ✅ Scenario 1: Onboarding Flow
* **전제**: 앱을 처음 설치한 상태
* **미션**:
    1. 앱을 실행하고 환영 화면을 확인한다.
    2. '목표 선택(Goal Selection)' 단계에서 카드 스와이프를 시도한다. (좌/우)
    3. 최소 1개 이상의 목표를 선택하고 온보딩을 완료한다.
* **확인 포인트**:
    * [ ] 목표 카드의 내용이 잘 읽히는가?
    * [ ] 스와이프 제스처가 끊김 없이 작동하는가?

### ✅ Scenario 2: Daily Logging (WriteView)
* **전제**: 온보딩 완료 후 홈 화면
* **미션**:
    1. 하단 `+` 버튼을 눌러 기록 화면(WriteSheet)을 연다.
    2. 오늘 느낀 감정과 활동을 카드로 선택한다.
    3. 시간대별 차트(TimeSlot)에서 드래그하여 에너지를 입력해본다.
    4. 저장을 완료하고 홈 화면(Railroad)에 반영되었는지 확인한다.
* **확인 포인트**:
    * [ ] 10분 단위 스냅이 의도대로 작동하는가?
    * [ ] 저장 직후 홈 화면이 즉시 갱신되는가?
